Implementation Decisions

  • Added a repo-level AGENTS.md after the user asked for incremental commit and message-format instructions.
  • Used mocked contact deltas and a mocked Photon Residency Dinner calendar event to prove the agent loop before native iOS work.
  • Kept the UI chat-first, with side panels exposing event state, candidates, and saved memories for demo clarity.
  • Added repo-owned docs for the product spec, demo plan, handoff context, and Codex access setup so future sessions do not depend on external planning files.
  • Imported the original Superpowers spec and implementation plan into docs/superpowers/ so the planning trail is versioned with the repo.
  • Iteration 2 adds relationship-agent core modules under src/relationship/ so the current UI demo remains stable while the agent architecture is built.
  • Changed TypeScript module resolution to Bundler so Vite can resolve Spectrum SDK package export subpaths such as spectrum-ts/providers/imessage.
  • The Spectrum adapter is a transport wrapper around the relationship agent core; the agent core remains usable through terminal tests without Spectrum credentials.
