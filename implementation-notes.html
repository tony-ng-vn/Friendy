Implementation Notes

  • Initial build target is a local web demo with mocked contact and calendar signals.
  • Native Contacts and Calendar integrations are intentionally deferred.
  • The app is implemented at the repository root because the Friendy repo was empty except for README.md.

Implementation Decisions

  • Added a repo-level AGENTS.md after the user asked for incremental commit and message-format instructions.
  • Used mocked contact deltas and a mocked Photon Residency Dinner calendar event to prove the agent loop before native iOS work.
  • Kept the UI chat-first, with side panels exposing event state, candidates, and saved memories for demo clarity.
  • Added repo-owned docs for the product spec, demo plan, handoff context, and Codex access setup so future sessions do not depend on external planning files.
  • Imported the original Superpowers spec and implementation plan into docs/superpowers/ so the planning trail is versioned with the repo.
  • Iteration 2 adds relationship-agent core modules under src/relationship/ so the current UI demo remains stable while the agent architecture is built.
  • Changed TypeScript module resolution to Bundler so Vite can resolve Spectrum SDK package export subpaths such as spectrum-ts/providers/imessage.
  • The Spectrum adapter is a transport wrapper around the relationship agent core; the agent core remains usable through terminal tests without Spectrum credentials.
  • Installing spectrum-ts reported high-severity npm audit warnings in transitive dependencies; no automatic audit fix was run because it could make unrelated dependency changes during the MVP pass.
  • Added explicit .env.local/.env loading for standalone tsx agent scripts because Vite env loading does not apply to npm run agent:spectrum.
  • Expanded manual memory capture so the live agent accepts natural first-person messages like I met Amaya at Photon Residency II... instead of only command-style met Name, context inputs.
  • Added docs/agent-navigation.md to document the repo navigation strategy: keep REFERENCE.md as the index, keep AGENTS.md files short and scoped, and avoid turning always-loaded instructions into copied specs.
  • Added an LLM interpretation layer for Friendy messages. OpenRouter is used only to produce validated intent JSON; deterministic relationship tools still execute all memory writes, searches, ignores, and clarification behavior.
  • Kept Spectrum/iMessage as a transport adapter by moving interpreted execution into a testable Spectrum runtime helper that returns reply text plus compact interaction logs.
